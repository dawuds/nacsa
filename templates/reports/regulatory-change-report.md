## 1. Purpose and Scope

### 1.1 Purpose

This Regulatory Change Impact Report documents the assessment of new or amended regulations, directives, or standards affecting the organization's compliance posture as a National Critical Information Infrastructure (NCII) entity under the **Cyber Security Act 2024 (Act 854)**. This report is prepared in accordance with the obligations set out under:

- **Section 15, Cyber Security Act 2024 (Act 854)** — Duties of NCII entity owners to comply with directions issued by the Chief Executive of NACSA
- **Section 24, Cyber Security Act 2024 (Act 854)** — Requirements for cybersecurity risk assessments and compliance reporting

## 5. Gap Analysis

*Compare the organization's current compliance posture against the requirements introduced by the regulatory change. For each identified gap, document the current state, the desired target state, and the remediation action required.*

### 5.1 Gap Analysis Methodology

*Describe the methodology used to identify and assess gaps. Reference any standards or frameworks used to structure the assessment (e.g., NIST CSF, ISO/IEC 27001, NACSA NCII Sector guidelines).*

The gap analysis was conducted using the following approach:

1. **Requirement Decomposition** — Each new or amended requirement (as documented in Section 2.3) was decomposed into discrete, testable compliance controls.
2. **Current State Assessment** — Existing controls, policies, and processes were reviewed against each requirement through [document review / interviews / technical testing / walkthrough].
3. **Gap Identification** — Deviations between the current state and the required target state were identified and documented.
4. **Gap Classification** — Each gap was classified by severity and type.
5. **Remediation Mapping** — Remediation actions were identified and assigned to responsible owners.

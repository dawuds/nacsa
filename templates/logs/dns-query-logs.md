## 1. Purpose and Scope

### 1.1 Purpose

This document defines the logging requirements, structure, and monitoring procedures for DNS (Domain Name System) query logs generated by National Critical Information Infrastructure (NCII) systems operated by [Organization Name]. DNS query logs are a mandatory operational security control under the **Cyber Security Act 2024 (Act 854)**, and serve as a primary data source for network-layer threat detection, incident investigation, and regulatory compliance demonstration.

These logs support continuous monitoring obligations under **Act 854, Section 18**, which requires NCII entities to implement and maintain security measures commensurate with cyber threats, including the detection of malicious network activity through system logging.

### 1.2 Scope

This template applies to:

- All DNS resolvers, forwarders, and recursive query servers within the NCII perimeter of [Organization Name]
- DNS queries originating from or destined to NCII-designated systems, endpoints, and servers
- Internal DNS infrastructure including split-horizon, stub zones, and authoritative name servers
- Third-party or cloud-managed DNS resolvers processing queries on behalf of [Organization Name] systems

This template **does not** apply to:

- DNS infrastructure wholly outside the NCII boundary with no integration to critical systems
- Public-facing authoritative DNS managed exclusively by external registrars without query-level visibility

## 3. DNS Queries and Responses

*Record all DNS queries issued by NCII systems and their corresponding resolver responses. Each log entry should capture sufficient detail to reconstruct query context, identify the requesting host, and assess the legitimacy of the resolution. Log fields should conform to the organisation's SIEM ingestion schema.*

### 3.1 Log Fields — Standard DNS Query Record

| Field Name | Data Type | Description | Example |
|---|---|---|---|
| `timestamp` | ISO 8601 DateTime | Time query was received by resolver | `2025-06-01T08:34:12.441Z` |
| `source_ip` | IPv4/IPv6 | IP address of the querying host | `10.10.5.42` |
| `source_port` | Integer | UDP/TCP source port | `54312` |
| `client_hostname` | String | Resolved hostname of querying host | `[Workstation/Server Name]` |
| `query_name` | FQDN | Domain name queried | `[example.domain.com]` |
| `query_type` | String | DNS record type requested | `A`, `AAAA`, `MX`, `TXT`, `PTR` |
| `query_class` | String | DNS query class | `IN` |
| `response_code` | String | DNS response code (RCODE) | `NOERROR`, `NXDOMAIN`, `SERVFAIL` |
| `resolved_ips` | List\<IP\> | IP addresses returned in the answer section | `[x.x.x.x]` |
| `ttl` | Integer (seconds) | Time-to-live of the response record | `300` |
| `resolver_ip` | IPv4/IPv6 | DNS resolver that processed the query | `[Resolver IP]` |
| `protocol` | String | Transport protocol used | `UDP`, `TCP`, `DoH`, `DoT` |
| `response_size` | Integer (bytes) | Size of the DNS response payload | `[Value]` |
| `query_duration_ms` | Integer | Query round-trip time in milliseconds | `[Value]` |

## 6. DNS Tunnelling Detection

*Log events where DNS queries or responses exhibit characteristics consistent with DNS tunnelling — the abuse of DNS as a covert data exfiltration or command-and-control channel. Indicators include unusually large TXT or NULL record payloads, high query rates to a single domain, and encoded data patterns in query names.*

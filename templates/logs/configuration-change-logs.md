## 1. Purpose and Scope

### 1.1 Purpose

This document establishes the framework and record structure for logging all configuration changes made to National Critical Information Infrastructure (NCII) system components within [Organization Name]. These logs serve as the authoritative audit trail for configuration state management, deviation detection, and compliance verification under the **Cyber Security Act 2024 (Act 854)**, administered by the **National Cyber Security Agency (NACSA)**.

Configuration Change Logs directly support compliance with **Section 18 of Act 854**, which mandates that NCII entities implement and maintain controls to protect, monitor, and respond to changes within designated critical systems. Continuous logging and daily review of configuration changes ensures that [Organization Name] can detect, investigate, and remediate unauthorised or non-compliant modifications in a timely manner.

### 1.2 Scope

This log framework applies to:

- All NCII-designated systems, platforms, and infrastructure components operated or managed by [Organization Name]
- Network devices (firewalls, routers, switches, load balancers)
- Server and endpoint operating system configurations
- Application and middleware configuration files
- Cloud service configurations and Infrastructure-as-Code (IaC) templates
- Security tooling configurations (SIEM, EDR, IDS/IPS, WAF)
- Identity and access management (IAM) policy configurations
- Database and storage system configurations

**Out of Scope:** End-user workstation settings not classified under NCII scope, unless otherwise designated by [Organization Name]'s NCII asset register.
